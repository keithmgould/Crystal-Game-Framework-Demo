This is a demo of the Crystal game framework. Having a realtime authorative server is a pain, especially in the face of an inertial system with no friction. So what better to demo with than ships flying through space? You are the little triangle, and you move the ship with the up, right and left arrows.

There are some knobs you can play with to the right:
  • extra latency: you can modify how much artificial latency the network encounters
  • interpolation delay: how far behind "right now" do we delay for interpolation.
  • final checkbox: shows the final results of Crystals algorithm.
  • server checkbox: shows the position of the server's snapshot.
  • future checkbox: fastforward of the server's snapshot
  • colorChange checkbox: show green when in predict mode, white when in snapshot interpolation mode
Crystal is a real-time "twitch" game framework with server and browser side components, written entirely in Javascript. It offers features normally found only in enterprise game engines, such as an authorative server, client side prediction, and position interpolation. Crystal is powered by a great 2D Physics engine which is run both on the server and client, allowing for a "zero-lag" experience. You can read more about Crystal's features in the repo's documentation. Crystal allows you to focus on your game, not the details of realizing a realtime experience in the face of lag.
